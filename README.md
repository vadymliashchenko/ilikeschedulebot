# Бот для збору статусів занять (студія танців)

## Що робить бот

- Щовечора (нд–чт, 21:00) запитує в групі хореографів статус завтрашніх занять кнопками
- О 22:00 нагадує тим, хто ще не відповів
- О 22:30 публікує готову таблицю в групі хореографів (закриті групи 🔒 показані статично)
- Веде облік лімітів "відмін/замін" (2 на місяць по групі) і пропущених опитувань (3 на місяць) — алерти в приватний чат власника
- В останній день місяця надсилає звірку по кожному хореографу в приватний чат
- Керування активним складом (➕ додати / ➖ прибрати / ✏️ змінити стиль чи час) — командою `/admin` у приватному чаті власника

## Крок 1. Створити бота і отримати токен

1. У Telegram знайдіть **@BotFather**, надішліть `/newbot`, дайте боту ім'я
2. BotFather видасть **токен** — довгий рядок на кшталт `123456789:AAExampleTokenXXXXXXXXXXXXXXXXXXXXX`
3. Збережіть цей токен — він піде в змінну `BOT_TOKEN`

## Крок 2. Додати бота у потрібні чати

1. Додайте бота **адміністратором** у вже існуючу групу хореографів
2. Створіть **новий приватний чат** з ботом (просто напишіть йому щось) — це буде чат власника
3. Клієнтську групу поки не чіпаємо — додамо бота туди пізніше, коли будете готові вмикати публікацію

## Крок 3. Дізнатися ID чатів

Найпростіший спосіб:

1. Відкрийте в браузері (замість `<TOKEN>` вставте свій токен):
   `https://api.telegram.org/bot<TOKEN>/getUpdates`
2. Напишіть будь-яке повідомлення в групі хореографів і окремо боту в приватному чаті
3. Оновіть сторінку з посиланням вище — у відповіді знайдете `"chat":{"id": -1001234567890, ...}` для групи (від'ємне число) і `"chat":{"id": 987654321, ...}` для приватного чату (додатне число)
4. Групові ID — це `CHOREO_GROUP_CHAT_ID`, приватний — це `OWNER_CHAT_ID`

## Крок 4. Налаштувати змінні середовища

Скопіюйте `.env.example` у файл `.env` і заповніть:

```
BOT_TOKEN=<ваш токен>
CHOREO_GROUP_CHAT_ID=<ID групи хореографів>
OWNER_CHAT_ID=<ID вашого приватного чату>
CLIENT_GROUP_CHAT_ID=
CLIENT_PUBLISHING_ENABLED=false
```

## Крок 5. Тестовий запуск локально (необов'язково)

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python -m bot.main
```

Якщо все налаштовано правильно, бот запуститься і почне чекати на розклад за годинником.

## Крок 6. Деплой на Railway (безкоштовно)

1. Створіть акаунт на [railway.app](https://railway.app)
2. Створіть новий проєкт → "Deploy from GitHub repo" (потрібно залити цю папку у GitHub-репозиторій) або скористайтесь Railway CLI (`railway up`) прямо з цієї папки
3. У розділі Variables проєкту додайте ті самі змінні, що й у `.env`
4. Railway сам підхопить `Procfile` і запустить бота як worker-процес
5. Перевірте логи в Railway — там буде видно `Scheduler started`, якщо все гаразд

## Що далі

- Поки триває тестовий режим — усе працює тільки в групі хореографів
- Коли переконаєтесь, що все стабільно — додайте бота в клієнтську групу, впишіть її ID в `CLIENT_GROUP_CHAT_ID` і поставте `CLIENT_PUBLISHING_ENABLED=true`
- Якщо захочете щось змінити (додати хореографа, поправити текст, час) — просто напишіть про це в чат, де ми це обговорювали, і зміни внесуться в цей самий код і передеплояться на цьому самому боті
